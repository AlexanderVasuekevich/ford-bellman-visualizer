# Визуализатор алгоритма Форда-Беллмана

Java Swing приложение для визуализации работы алгоритма Форда-Беллмана на
ориентированном взвешенном графе.

Программа позволяет загрузить граф из файла, проверить корректность входных
данных, отобразить граф, построить пошаговую историю алгоритма, переходить по
шагам вперед и назад, запускать автоматический режим и сохранять результат в
текстовый файл.

## Требования

- JDK 25 или совместимая версия JDK.
- Терминал с доступом к командам `javac` и `java`.

## Сборка и запуск

Из корня репозитория выполнить команду

```bash
javac -d /tmp/ford-bellman-classes $(find src/main/java -name '*.java')
java -cp /tmp/ford-bellman-classes visualizer.Main
```

Скомпилированные `.class` файлы в этом варианте кладутся в:

```txt
/tmp/ford-bellman-classes
```

## Запуск тестов

Из корня репозитория выполнить команды

```bash
javac -d /tmp/ford-bellman-test-classes \
  $(find src/main/java tests -name '*.java')
java -cp /tmp/ford-bellman-test-classes parser.GraphFileParserTest
java -cp /tmp/ford-bellman-test-classes algorithm.BellmanFordAlgorithmTest
```

`GraphFileParserTest` проверяет чтение и валидацию входных файлов.
`BellmanFordAlgorithmTest` — комплексное тестирование алгоритма: отрицательные
циклы, недостижимые вершины, большие графы (10+ вершин), навигация по истории
шагов, экспорт результата и сценарий ручного редактирования.

## Формат входного файла

Граф задается текстовым файлом. Пустые строки допускаются.

Поддерживаемые команды:

```txt
SOURCE A
VERTEX A
EDGE A B 10
```

Где:

- `SOURCE A` - стартовая вершина алгоритма.
- `VERTEX A` - объявление вершины `A`.
- `EDGE A B 10` - ориентированное ребро из `A` в `B` с весом `10`.

Пример корректного файла:

```txt
SOURCE A

VERTEX A
VERTEX B
VERTEX C
VERTEX D

EDGE A B 4
EDGE A C 5
EDGE B C -2
EDGE C D 3
```

Координаты вершин во входном файле не задаются. При загрузке из файла
расположение вершин рассчитывается автоматически.

## Тестовые данные

Корректные графы находятся в:

```txt
test-data/valid
```

Файлы с ошибками для проверки валидации находятся в:

```txt
test-data/invalid
```

## Ручное редактирование графа

Режим редактирования включается кнопкой `Редактировать` на панели управления.
Поддерживается:

- создание вершины по клику на свободное место в режиме редактирования;
- открытие окна редактирования по клику на вершину;
- изменение имени вершины;
- добавление ребра командой `EDGE A B 100` (Enter — применить);
- назначение текущей вершины стартовой командой `SOURCE`;
- удаление ребра через контекстное меню списка ребер (правый клик);
- удаление вершины кнопкой `Удалить`;
- кнопки `Сохранить`, `Удалить`, `Отмена` (или Ctrl+S, Ctrl+Delete, Esc).

Изменения в окне редактирования сначала применяются к черновику. Фактический граф
меняется только после сохранения. После изменения графа история шагов сбрасывается
и алгоритм нужно запустить заново.

## Перемещение по графу

Граф можно перетаскивать внутри области отображения, зажав среднюю кнопку мыши.
Если граф не помещается в окно целиком, доступны ползунки прокрутки.

## Финальный шаг алгоритма

На последнем шаге в области пояснения выводятся восстановленные кратчайшие
пути от стартовой вершины до каждой достижимой вершины. При обнаружении
отрицательного цикла выводится перечень вершин, расстояние до которых
не определено.

## Сохранение результата

После запуска алгоритма результат можно сохранить в текстовый файл через кнопку
`Сохранить в файл`.

Если файл уже существует, программа запрашивает подтверждение перезаписи.
Ошибки сохранения выводятся через модальное окно.
