# Спецификация проекта

## Визуализатор алгоритма Форда–Беллмана

## 1. Назначение программы

Программа предназначена для визуализации работы алгоритма Форда–Беллмана на ориентированном взвешенном графе.

Алгоритм Форда–Беллмана используется для поиска кратчайших расстояний от выбранной стартовой вершины до всех остальных вершин графа. Программа должна показывать пользователю не только итоговый результат, но и сам процесс работы алгоритма по небольшим шагам.

Приложение разрабатывается на языке Java с графическим интерфейсом.

## 2. Основные возможности программы

Программа должна поддерживать следующие возможности:

1. Загрузка графа из файла.
2. Проверка корректности входного файла.
3. Графическое отображение графа.
4. Отображение названий вершин, рёбер, их направлений и весов.
5. Запуск алгоритма Форда–Беллмана.
6. Пошаговая визуализация работы алгоритма.
7. Переход к следующему шагу алгоритма.
8. Возврат к предыдущему шагу алгоритма.
9. Автоматическое выполнение шагов с заданным интервалом.
10. Остановка автоматического выполнения.
11. Вывод текстовых пояснений к каждому шагу.
12. Отображение таблицы текущих расстояний.
13. Сохранение результата работы алгоритма в отдельный файл.
14. Обработка некорректных входных данных.
15. Окно `О разработчиках`.

## 4. Ввод данных

Граф можно задать двумя способами: загрузить из файла или создать вручную в интерфейсе.

При загрузке из файла, программа загружает заранее подготовленный файл с описанием графа, после чего происходит обязательная проверка его корректности.

Входной файл должен содержать:

- стартовую вершину
- список вершин
- список ориентированных взвешенных рёбер

Пустые строки допускаются.

### Пример входного файла

```text
SOURCE A

VERTEX A
VERTEX B
VERTEX C
VERTEX D

EDGE A B 4
EDGE A C 5
EDGE B C -2
EDGE C D 3
```

Где:

```text
SOURCE A     — стартовая вершина
VERTEX A     — объявление вершины A
EDGE A B 4   — ориентированное ребро из A в B с весом 4
```

## 5. Проверка входного файла

Программа должна проверять корректность файла.
Должны обрабатываться следующие ситуации:

1. Файл пустой.
2. Файл не существует или его невозможно открыть.
3. Файл имеет неподходящий формат.
4. Не указана стартовая вершина.
5. Стартовая вершина не объявлена в списке вершин.
6. Граф не содержит вершин.
7. Ребро ссылается на несуществующую вершину.
8. Вес ребра не является числом.
9. Есть повторяющиеся вершины.
10. Есть некорректная команда.
11. Строка файла имеет неверное количество аргументов.

При ошибке программа не должна аварийно завершаться. Она должна показать пользователю понятное сообщение об ошибке.

Пример сообщения:

```text
Ошибка в строке 7: вершина X не объявлена.
```

Для подобных сообщений нужно организовать модальное окно, которое будет переиспользоваться с параметрами:

```text
String title, String message
```

Это же модальное окно в дальнейшем может использоваться для окна `О разработчиках`.

## 6. Ручной ввод графа в приложении

Помимо загрузки графа из файла, программа должна поддерживать упрощённый режим редактирования графа.

Режим редактирования включается кнопкой `+` или `Редактировать`.

В обычном режиме клики по вершинам и рёбрам не изменяют граф. В режиме редактирования пользователь может создавать вершины и редактировать список рёбер.

### Создание вершины

В режиме редактирования пользователь нажимает левой кнопкой мыши по свободному месту в области графа.

После этого на холсте создаётся новая вершина, и открывается модальное окно создания вершины.

В окне создания вершины отображаются:

- поле `Name`;
- имя вершины по умолчанию, например `V11`, если в графе уже есть 10 вершин;
- список рёбер, связанных с вершиной;
- поле для добавления ребра в формате `EDGE A B 10`;
- кнопка `Сохранить`;
- кнопка `Удалить`.

Если пользователь нажимает `Сохранить`, вершина сохраняется в графе.

Если пользователь нажимает `Удалить`, создание вершины отменяется.

### Редактирование существующей вершины

Если пользователь находится в режиме редактирования и нажимает на уже существующую вершину, открывается такое же модальное окно.

В этом окне отображаются:

- текущее имя вершины;
- список рёбер, связанных с этой вершиной;
- поле для добавления нового ребра;
- возможность удалить ребро из списка;
- кнопка `Сохранить`;
- кнопка `Удалить`.

## 7. Пользовательский интерфейс

Главное окно программы должно содержать:

1. Панель управления с кнопками.
2. Область отображения графа.
3. Таблицу текущих расстояний.
4. Область текстового пояснения текущего шага.
5. Текущий проход
6. Текущий шаг

Если граф слишком большой и полностью не помещается в окно, должна быть возможность перемещаться по области графа с помощью ползунков.

Если таблица расстояний содержит много строк, она должна иметь отдельную вертикальную прокрутку.

### Взаимодействие с помощью мыши

Граф можно перетаскивать внутри заданного окна с помощью мыши, если зажата средняя кнопка мыши.

## 8. Графическое отображение графа

После успешной загрузки файла граф должен отображаться в окне программы.

На графе должны быть видны:

1. Все вершины.
2. Все рёбра.
3. Направления рёбер.
4. Веса рёбер.
5. Названия вершин.

Текущие элементы, участвующие в шаге алгоритма, должны выделяться цветом:

- текущее рассматриваемое ребро;
- начальная вершина рассматриваемого ребра;
- конечная вершина рассматриваемого ребра;
- вершина, расстояние до которой было обновлено.

## 9. Запуск алгоритма

После загрузки графа пользователь должен иметь возможность запустить алгоритм Форда–Беллмана.

Для этого используются кнопка `Шаг вперёд` для ручного режима и переключатель `Авто` для автоматического режима.

В альфа-версии допускается сразу выводить финальный результат работы алгоритма.
В финальной и бета-версии алгоритм должен выполняться пошагово.

## 10. Пошаговая визуализация алгоритма

Визуализация должна быть пошаговой. Один основной шаг визуализации соответствует рассмотрению одного ребра алгоритмом.

На каждом шаге пользователь должен видеть:

1. Номер текущего шага.
2. Номер текущего прохода алгоритма.
3. Какое ребро сейчас рассматривается.
4. Вес рассматриваемого ребра.
5. Текущие значения расстояний.
6. Проверяемое условие релаксации.
7. Было ли обновлено расстояние.
8. Если расстояние обновлено — какое значение изменилось.
9. Текстовое пояснение происходящего.

## 11. Пример текстового пояснения шага

Если расстояние обновилось:

```text
Проход 1 из 3.
Шаг 7.

Рассматривается ребро B → C с весом -2.

dist[B] = 4
dist[C] = 5

Проверка:
4 + (-2) = 2

2 < 5, расстояние до вершины C обновляется.
Новое значение dist[C] = 2.
Предыдущая вершина для C = B.
```

Если обновления нет:

```text
Шаг 8.
Проход 1 из 3.

Рассматривается ребро C → D с весом 3.

dist[C] = 2
dist[D] = 5

Проверка:
2 + 3 = 5

Так как 5 не меньше 5, расстояние до вершины D не изменяется.
```

## 13. История шагов

Для реализации перехода вперёд и назад программа должна хранить историю состояний алгоритма.

Каждый шаг алгоритма сохраняется как отдельное состояние.

Одно состояние содержит:

- номер шага;
- номер прохода;
- текущее ребро;
- текущие расстояния до всех вершин;
- предыдущие вершины;
- текстовое пояснение;
- информацию о том, произошло ли обновление;
- информацию об отрицательном цикле, если он обнаружен.

Благодаря этому:

- кнопка `Вперёд` показывает следующий шаг;
- кнопка `Назад` показывает предыдущий шаг;
- кнопка `Авто` автоматически переключает шаги вперёд с заданным интервалом.

## 13. Управление алгоритмом

В интерфейсе должны быть следующие кнопки.

### 13.1. Кнопка `Запустить`

Запускает подготовку шагов алгоритма.

После нажатия программа формирует историю шагов и показывает начальное состояние.

### 13.2. Кнопка `Вперёд`

Переход к следующему шагу алгоритма.

При переходе вперёд должны обновляться:

- подсветка на графе;
- таблица расстояний;
- текстовое описание текущего шага;
- номер текущего шага.

### 13.3. Кнопка `Назад`

Переход к предыдущему шагу алгоритма.

При переходе назад должны восстанавливаться:

- состояние графа;
- таблица расстояний;
- текстовое описание шага;
- номер текущего шага.

### 13.4. Кнопка-переключатель `Авто`

Запускает или останавливает автоматическое выполнение шагов.

### 13.6. Поле интервала

Поле ввода интервала задаёт задержку между шагами автоматического выполнения в миллисекундах.

Пример:

```text
1000
```

означает один шаг в секунду. Значение обновляется при каждом пользовательском вводе с небольшим debounce.

### 13.7 Окно `О разработчиках`

В программе должна быть кнопка `О разработчиках`.

При нажатии открывается модальное окно с информацией о команде.

В окне должны быть:

- ФИО участников с их группами;
- фотография и/или любимый мем.

### 13.8 Кнопка `Редактировать`

В программе должна быть кнопка `Редактировать`.

При нажатии включается режим редактирования графа. В этом режиме:

- При клике по вершине открывается окно с её редактированием.
- При клике не по существующим вершинам создаётся новая вершина.

## 14. Таблица текущих расстояний

В интерфейсе должна отображаться таблица текущих расстояний со столбцами:

- `Vertex` — вершина;
- `Distance` — текущее расстояние от стартовой вершины;
- `Parent` — предыдущая вершина в кратчайшем пути.

Если вершина недостижима, в поле `Distance` отображается `INF`.

Если строк много, таблица должна иметь вертикальную прокрутку с помощью ползунка.

## 15. Финальный результат алгоритма

После завершения алгоритма программа должна показать итоговые расстояния от стартовой вершины до всех остальных вершин.

Финальный результат должен содержать:

- стартовую вершину
- расстояние до каждой вершины
- предыдущую вершину для каждой вершины
- восстановленный путь для каждой вершины
- информацию о недостижимых вершинах
- информацию об отрицательном цикле, если он найден

Пример результата:

```text
Источник: A

A: расстояние = 0, путь = A
B: расстояние = 4, путь = A → B
C: расстояние = 2, путь = A → B → C
D: расстояние = 5, путь = A → B → C → D
```

Если вершина недостижима:

```text
E: недостижима
```

Если найден отрицательный цикл:

```text
Обнаружен отрицательный цикл, достижимый из стартовой вершины.
Кратчайшие пути не могут быть корректно определены.
```

## 16. Формат выходного файла

Пользователь должен иметь возможность сохранить результат работы алгоритма в отдельный файл.

Пример выходного файла:

```text
Алгоритм: Форда–Беллмана
Источник: A
Статус: успешно завершено

Вершина; Расстояние; Предыдущая вершина; Путь
A; 0; -; A
B; 4; A; A -> B
C; 2; B; A -> B -> C
D; 5; C; A -> B -> C -> D
```

Если есть отрицательный цикл:

```text
Алгоритм: Форда–Беллмана
Источник: A
Статус: обнаружен отрицательный цикл

Кратчайшие пути не могут быть корректно определены.
```

## 17. Обработка ошибок

Программа не должна аварийно завершаться при некорректных действиях пользователя или неверных файлах.

Должны обрабатываться следующие ситуации:

1. Файл пустой.
2. Файл невозможно открыть.
3. Файл имеет неподходящий формат.
4. Не указана стартовая вершина.
5. Стартовая вершина не объявлена.
6. Граф не содержит вершин.
7. Ребро ссылается на несуществующую вершину.
8. Вес ребра не является числом.
9. Вершина объявлена повторно.
10. Пользователь пытается запустить алгоритм без загруженного графа.
11. Пользователь пытается сохранить результат до выполнения алгоритма.
12. Пользователь вводит некорректный интервал автоматического выполнения.
13. Результат невозможно сохранить.

Ошибки должны показываться через модальное окно.

Модальное окно должно принимать параметры:

```text
title
message
```

Пример:

```text
title: Ошибка загрузки файла
message: Ошибка в строке 7: вершина X не объявлена.
```

## 18. Требования к репозиторию

В репозитории должны находиться:

- исходный код программы;
- спецификация;
- план работ;
- README;
- тестовые входные файлы;
- изображения или материалы для окна `О разработчиках`;
- материалы прототипа.

При обновлении репозитория нужно добавлять осмысленные комментарии к изменениям.

Примеры сообщений:

```text
Добавлена спецификация проекта
Добавлен план реализации проекта
Добавлена структура классов Graph, Vertex, Edge
Добавлен парсер входного файла
Исправлена ошибка обработки пустого файла
Добавлено окно "О разработчиках"
```

В последней версии репозитория не должно быть устаревших копий файлов.
