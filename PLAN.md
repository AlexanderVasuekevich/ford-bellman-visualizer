## Alpha-версия

Цель: загрузка графа из файла, отображение графа на экране, вывод финального результата алгоритма одной кнопкой.

### Васюкевич Александр

1. Создать структуру проекта (пакеты, точка входа `Main.java`).
2. Реализовать главное окно приложения (JFrame): разметка областей — панель управления сверху, холст графа по центру, таблица расстояний справа, текстовое пояснение снизу.
3. Реализовать кнопку «Загрузить файл» с `JFileChooser`.
4. Реализовать переиспользуемое модальное окно `MessageDialog(String title, String message)` для вывода ошибок.
5. Реализовать кнопку «Запустить» — вызов алгоритма и отображение финального результата в текстовой области.
6. Связать модули: парсер → модель графа → отрисовка → алгоритм.

### Бурменский Алексей

1. Реализовать классы модели данных: `Graph`, `Vertex`, `Edge`.
2. Реализовать `GraphParser` — чтение входного файла, создание объекта `Graph`.
3. Реализовать валидацию входного файла (пункты 1–11 из спецификации, раздел 5): пустой файл, несуществующий файл, неверный формат, отсутствие стартовой вершины, необъявленная стартовая вершина, отсутствие вершин, ребро к несуществующей вершине, нечисловой вес, дубликаты вершин, неизвестная команда, неверное число аргументов.
4. Подготовить набор тестовых `.txt` файлов: корректные графы (3–4 шт.), файлы с ошибками для каждого типа валидации.

### Стрижков Иван

1. Реализовать `GraphPanel` (JPanel) — отрисовка графа: вершины как круги с именами, рёбра как линии со стрелками и весами.
2. Реализовать автоматическое размещение вершин на холсте (например, по кругу или силовой алгоритм).
3. Реализовать алгоритм Форда–Беллмана (`BellmanFord.java`) — вычисление финальных расстояний и предшественников.
4. Реализовать статическую таблицу расстояний (JTable): столбцы Vertex, Distance, Parent.

**Результат Alpha:** приложение запускается, загружает файл, показывает граф, по кнопке «Запустить» показывает итоговые расстояния в таблице и текстовой области.

## Beta-версия

Цель: пошаговая визуализация алгоритма с навигацией вперёд/назад, автоматический режим, текстовые пояснения к каждому шагу.

### Васюкевич Александр

1. Реализовать кнопки «Вперёд», «Назад» — переключение между шагами алгоритма.
2. Реализовать кнопку-переключатель «Авто» — запуск/остановка автоматического выполнения через `javax.swing.Timer`.
3. Реализовать поле ввода интервала (мс) с debounce.
4. Реализовать обновление всех компонентов UI при переходе между шагами: подсветка на графе, таблица, текст, номер шага/прохода.
5. Реализовать окно «О разработчиках» — модальное окно с ФИО, группами, фотографиями/мемами.

### Бурменский Алексей

1. Реализовать класс `StepState` — хранение состояния одного шага: номер шага, номер прохода, текущее ребро, массив расстояний, массив предшественников, текстовое пояснение, флаг обновления, флаг отрицательного цикла.
2. Реализовать генерацию полной истории шагов (`List<StepState>`) при запуске алгоритма.
3. Реализовать генерацию текстовых пояснений к каждому шагу по формату из спецификации (раздел 11).
4. Реализовать детекцию отрицательного цикла (N-й проход) и соответствующее пояснение.
5. Реализовать сохранение результата алгоритма в файл (`JFileChooser` для выбора пути, формат из раздела 16).

### Стрижков Иван

1. Реализовать подсветку текущего ребра и вершин на графе при переходе между шагами: разные цвета для текущего ребра, начальной вершины, конечной вершины, обновлённой вершины.
2. Реализовать динамическое обновление таблицы расстояний при каждом шаге.
3. Реализовать отображение номера текущего шага и номера прохода в интерфейсе.
4. Реализовать прокрутку области графа (JScrollPane) при больших графах.
5. Реализовать прокрутку таблицы расстояний при большом числе вершин.

**Результат Beta:** алгоритм выполняется пошагово, пользователь переключает шаги вперёд/назад, видит подсветку, текстовые пояснения и обновление таблицы. Работает авто-режим.

## Final-версия

Цель: ручной ввод графа в приложении, перетаскивание графа мышью, восстановление путей, полная обработка ошибок, полировка.

### Васюкевич Александр

1. Реализовать кнопку «Редактировать» — включение/выключение режима редактирования.
2. Реализовать создание вершины по клику на свободное место в режиме редактирования.
3. Реализовать модальное окно создания/редактирования вершины: поле имени, список рёбер, поле добавления ребра (`EDGE A B 10`), кнопки «Сохранить» и «Удалить».
4. Финальная полировка UI: выравнивание элементов, состояния кнопок (enabled/disabled), единообразный стиль.

### Бурменский Алексей

1. Реализовать логику добавления/удаления вершин и рёбер из модели графа через интерфейс редактирования.
2. Реализовать обработку оставшихся ошибок (пункты 10–13 из раздела 17): запуск без графа, сохранение до выполнения, некорректный интервал, невозможность сохранения файла.
3. Написать README.md для репозитория: описание проекта, инструкция по сборке и запуску, описание формата файла.
4. Финальное тестирование всех сценариев валидации.

### Стрижков Иван

1. Реализовать перетаскивание графа средней кнопкой мыши.
2. Реализовать отображение восстановленных путей в финальном результате (раздел 15).
3. Комплексное тестирование: графы с отрицательными циклами, недостижимые вершины, большие графы (10+ вершин), крайние случаи.
4. Исправление багов, найденных при тестировании.

**Результат Final:** полнофункциональное приложение — загрузка из файла и ручной ввод графа, пошаговая визуализация с пояснениями, навигация, авто-режим, сохранение результата, обработка всех ошибок.
